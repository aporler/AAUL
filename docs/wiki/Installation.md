# Installation

Note: This wiki is valid for version 0.10 (January 19, 2026).

## English

This page covers installing the dashboard and running the two servers:

- API server (Express)
- UI server (Vite dev server)

### Option A: Automated install (recommended for dev)

From the project root:

```
chmod +x install.sh
./install.sh
```

What it does:

- Detects the OS (Linux or macOS)
- Installs Node.js 18+ if missing
- Installs build tools required by native Node modules
- Creates `.env` files from examples if missing
- Installs dependencies for both dashboard and client
- Builds the agent bundle
- Starts both servers with `npm run dev:all`

### Option B: Manual install

Requirements:

- Node.js 18+
- Python 3 (agents only)
- systemd on Linux, launchd on macOS, Task Scheduler on Windows (agents only)

Steps:

```
cd dashboard
npm install

cd client
npm install

cd ..
npm run dev:all
```

Open the UI at `http://localhost:5173`.

### Production notes

1) Build the UI:

```
cd dashboard/client
npm install
npm run build
```

2) Build the agent bundle:

```
bash dashboard/scripts/build-agent-bundle.sh
```

3) Start the API server:

```
cd dashboard
npm install
npm run start
```

The API serves the UI in production from `dashboard/client/dist`.

Agent installers generated by the dashboard now download the platform script locally first, then execute it with the right shell (`sh`, `bash`, `PowerShell`) instead of using a direct `curl | sh` pattern.
